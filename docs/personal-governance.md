# Personal Governance Model v2

基于 Ordivon 方法论的个人成长治理系统。Credit: 初始七层模型由 Hermes Agent 提出，核心修正由用户完成。

## 八层架构

```
Intent   ─→ 决定优化方向：求真、成为、分享、贡献
    │
Identity ─→ 初始约束与筛选条件，不等于命运
    │
Health   ─→ 每日能量预算，所有系统的底盘
    │
Mind     ─→ 判断/学习/情绪控制/风险控制，核心控制器
    │
Skill    ─→ 把认知转化为可执行能力，必须由项目和任务验证
Social   ─→ 把 Output 转化为信任、机会、协作和分发
Capital  ─→ 时间、现金、风控和行动选项
    │
Output   ─→ 外部 Receipt，系统获得反馈和信用的唯一接口
    │
Feedback ─→ 反向更新 Mind / Skill / Social / Capital
```

## 关键修正

### 1. Identity ≠ 命运

| 项目 | 分类 |
|------|------|
| 年龄 | 不可逆约束 |
| 国籍/户籍 | 高成本约束 |
| 学校标签 (普通一本) | 初始筛选标签，可被 Output 部分覆盖 |
| 家庭支持度 | Capital + Social + Psychological Safety |
| 应届/非应届 | 时间窗口状态，不是永久身份 |

**治理规则:** 不做短期主战场。承认筛选影响，用 Output、Skill、Social 生成反证。别人有清北/姚班标签，你需要 GitHub、文章、项目、公开记录来替代。

### 2. Capital ≠ 交易入口

OKX 账户只是市场接入权限。真正的 Capital 资产:

- 本金
- 风控系统 (可验证的规则，不是感觉)
- 交易纪律 (执行一致性)
- 可验证正期望策略 (OOS 验证)
- 最大回撤控制
- 长期存活能力

**治理规则:** 没有稳定正期望之前，交易不是财富引擎，是认知训练场 + 风险暴露源。大一阶段用小仓位学习，主资源投入 Skill、Mind、Output。

### 3. Output ≠ 自我证明

Output 是**外部可验证记录**，不是"我很强"的声明。

- 无 Receipt = 无外部证据 = 不产生社会信用
- 别人看不到你的思考密度/焦虑/潜力/方法论
- 只能看到: 文章、项目、代码、PR、复盘、表达、比赛、实习

**治理规则:** 每周至少一个外部 Receipt。

### 4. Skill ≠ 学过什么

| 不严格 | 严格 |
|--------|------|
| "我会 Rust" | 能用 Rust 写带测试、CLI、错误处理、文档的小工具 |
| "我会 Python" | 能用 Python 做数据清洗、回测、可视化、自动化脚本 |
| "我懂 AI Agent" | 能设计带状态、工具调用、日志、失败恢复的 agent harness |
| "我懂交易系统" | 能实现策略→回测→风控→记录→复盘闭环 |

**指标:** 任务完成记录 + 可复现项目 + 他人可用性 + 真实反馈。不是 GitHub contributions 数量。

### 5. Social ≠ 等作品自动吸引

真实公式: **可验证作品 + 稳定表达 + 互惠互动 + 长期信用 = 社会资本**

不需要"社交牛逼症"，但也不能完全被动。正确做法: 小作品公开化，小问题认真请教，小贡献持续给出，小范围建立信用。

## 每周七问 (可执行版)

```text
1. Intent:   我有没有偏离"求真、成为、分享、贡献"？
2. Identity: 我有没有在抱怨短期无法改变的东西？
3. Health:   我的睡眠、运动、精力有没有低于及格线？
4. Mind:     我这周有没有过载、逃避、情绪化决策？
5. Skill:    我有没有真正完成一个可验证任务？
6. Social:   我有没有让别人看到一个真实进展？
7. Capital:  我的时间和钱有没有被错误下注？
8. Output:   这周有没有留下一个外部 Receipt？
```

## 当前状态评估 (2026-05-30)

**最大资产:** 18岁 + 高好奇心 + 强系统化倾向 + 低生活成本 + 四年试错窗口

**最大负债:** 看太多、想太多、系统化太多、对外产出太少、健康习惯未固化

**当前优先级:**

1. Health 先达标 (身体不及格时不上高杠杆交易、不疯狂开新项目)
2. Skill 必须外部化 (每学一个东西 → 项目/文章/复盘/PR)
3. Output 每周最小化交付 (不等大作品，建输出肌肉)
