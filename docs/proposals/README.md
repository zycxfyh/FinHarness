# Proposals

Use this folder for substantial new layers, workflow redesigns, or experiments
that should be clarified before implementation.

A proposal is not a long specification. It is a pre-action reasoning record.

## When To Write One

Write a proposal when the change affects:

```text
layer boundaries
data schemas
AI agent authority
risk gates
execution paths
portfolio or broker behavior
cross-module workflows
```

Do not write one for small local implementation details.

## Template

```text
# Proposal: <Title>

Date:
Status: draft | accepted | testing | rejected | superseded
Related idea:
Related module:
Related ADR:

## Problem

What problem are we solving?

## User / Workflow

Who or what benefits?

## Goals

What should become possible?

## Non-Goals

What are we explicitly not solving?

## Evidence

What sources, notes, experiments, or receipts support this?

## Design

What is the proposed shape?

## Inputs / Outputs

Typed inputs:
Typed outputs:

## Quality / Lineage / Receipt

How will this be auditable?

## Risks

What can go wrong?

## Success Signal

How will we know it worked?

## Review Plan

When and how will we review the outcome?
```
