# FinHarness Evolution Roadmap

> **Documentation lifecycle:** `historical`
> **Current authority:** [FinHarness Current System](../current-system.md) and [Program #277](https://github.com/zycxfyh/FinHarness/issues/277)
> **Reason:** This maintained Markdown roadmap duplicated GitHub work state, Program sequencing, the system catalog, debt counts, and historical PR evidence. It was retired during Lean Governance P1.

This path is retained only to explain the transition. The complete authored
roadmap remains available in Git history at
`df0f4317ca6194aaf56ecd4f92c0a9b5e64f4cb2` and earlier commits.

Current responsibilities are intentionally separated:

- current capability, boundaries, and fact ownership:
  [`docs/current-system.md`](../current-system.md);
- current product objective and investment intent:
  [GitHub Program #277](https://github.com/zycxfyh/FinHarness/issues/277);
- mutable work authorization and sequence:
  GitHub Issue/PR state, labels, and native relationships;
- system ownership and lifecycle:
  [`system-catalog.yml`](system-catalog.yml);
- verified engineering debt:
  [`debt-register.json`](../governance/debt-register.json);
- historical implementation reasoning:
  Git history, ADRs, proposals, reviews, and notes.

Do not restore a maintained Markdown implementation sequence. A future roadmap
is justified only if GitHub-native work state cannot express an observed,
material need and the new artifact replaces an existing authority rather than
adding another one.
