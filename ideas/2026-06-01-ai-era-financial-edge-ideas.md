# Ideas: AI-Era Financial Edge

Date: 2026-06-01

## IDEA-001: Financial Decision OS Edge

Idea:
Build FinHarness around the full financial decision loop, not around a trading
bot or a single prediction model.

Hypothesis:
In AI-era finance, durable edge comes from making the loop from information to
judgment to action to feedback faster, cleaner, safer, and more auditable.

Minimum experiment:
Define a typed workflow that connects:

```text
MarketDataSnapshot
-> IndicatorSnapshot
-> EventSnapshot
-> Interpretation
-> Hypothesis
-> Validation
-> Proposal
-> RiskGate
-> ExecutionReceipt
-> Review
```

Success signal:
One paper trade idea can be reconstructed from source evidence, intermediate
interpretation, proposal, risk decision, execution/rejection, and review without
reading chat logs.

## IDEA-002: AI-Era Edge Scorecard

Idea:
Score each FinHarness workflow by the kind of edge it creates.

Hypothesis:
Many AI-finance products look impressive but only create surface edge. A
scorecard can force the project to distinguish UI edge, speed edge, evidence
edge, process edge, behavior edge, and feedback edge.

Minimum experiment:
Add a review template that scores a workflow on:

```text
information speed
information coverage
evidence quality
interpretation quality
validation speed
risk discipline
behavior control
feedback compounding
```

Success signal:
New features can be rejected if they are visually impressive but do not improve
the decision loop.
